## Accomplishments
- Created `backend/app/local/` module with LocalMachineManager that auto-detects Docker vs native mode and provides SSH or subprocess transport accordingly
- Built local tmux operations (list/check/create/kill) via subprocess, mirroring SSH tmux function signatures for consistency
- Implemented LocalTerminalProcess with full duck-typing (stdout/stdin/channel interfaces) so SessionProcess in ws/terminal.py works without any code changes
- Created unified machine registry in services/machine_registry.py that routes all machine lookups by machine_id
- Relaxed TerminalSession.machine_id from UUID FK to plain String(255) to support machine_id="local"
- Added automated startup migration that converts existing UUID columns to VARCHAR(255) and drops the FK constraint
- Configured Docker host.docker.internal resolution and local SSH env vars for both app and app-dev services

## Decisions Made
- Used AliasChoices + raw string property pattern for `local_repo_scan_paths` because pydantic-settings v2 JSON-parses `list[str]` fields from env vars before field validators run, making comma-separated parsing impossible with a validator alone
- Made `LocalTerminalProcess.spawn()` a classmethod factory instead of inlining fork logic in tmux.py -- cleaner separation of PTY creation from tmux session management
- Used inline startup migration (ALTER TABLE in lifespan) instead of Alembic for the single schema change -- runs every startup but is a no-op when column is already VARCHAR

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Fixed pydantic-settings list parsing for env vars**
- **Found during:** Task 1 (config.py local_repo_scan_paths)
- **Issue:** pydantic-settings v2 tries to JSON-parse `list[str]` fields from env vars before field_validator runs, causing JSONDecodeError on comma-separated values
- **Fix:** Changed to `local_repo_scan_paths_raw: str` with AliasChoices and a `@property` that parses the comma-separated string into a list
- **Files modified:** backend/app/config.py
- **Verification:** Tested with `LOCUS_LOCAL_REPO_SCAN_PATHS="path1,path2"` and empty default
- **Committed in:** 6ef17a0 (Task 1 commit)
