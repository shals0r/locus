- GET /api/machines returns "This Machine" as first entry with id="local" and status="online"

- Terminal WebSocket routes local sessions through PTY (native) or SSH (Docker) transparently
- Status WebSocket includes local machine in initial snapshot and all polling cycles
- Subprocess-based Claude session detection for native mode matches SSH version interface

## Decisions Made
- CRUD/connect/disconnect endpoints keep `machine_id: UUID` signatures -- "local" correctly fails UUID validation for remote-only operations, which is the desired behavior
- Death animation is skipped for local native mode (when `conn is None`) because subprocess-based tmux pane TTY injection adds complexity without essential value
- Subprocess-based Claude detection mirrors the SSH version exactly for consistent status output parsing
