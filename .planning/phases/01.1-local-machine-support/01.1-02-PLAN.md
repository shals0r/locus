**CRITICAL: UUID-to-str endpoint signature changes.** Multiple endpoints in `machines.py` currently use `machine_id: UUID` as a FastAPI path parameter type. FastAPI validates path parameters BEFORE the handler runs, so `machine_id: UUID` will reject `machine_id="local"` with a 422 error. ALL endpoint signatures that accept a machine_id path parameter must be changed from `machine_id: UUID` to `machine_id: str`. Then, on the remote-machine code path inside each handler, explicitly convert to UUID for DB lookups: `UUID(machine_id)`.

**`api/machines.py`**: Modify the machine list and related endpoints to include "This Machine":

1. Add imports: `from app.local.manager import LOCAL_MACHINE_ID, LOCAL_MACHINE_NAME, local_machine_manager` and `from app.services.machine_registry import is_local_machine, get_connection_for_machine` and `from app.local.tmux import list_tmux_sessions_local, create_local_terminal_in_tmux`.

2. Create a helper function `_local_machine_response()` that returns a `MachineResponse` for the local machine:
   - `id=LOCAL_MACHINE_ID` (the string "local")
   - `name=LOCAL_MACHINE_NAME` ("This Machine")
   - `host="localhost"`, `port=0`, `username=""`, `ssh_key_path=""`
   - `repo_scan_paths=settings.local_repo_scan_paths` (from config)
   - `status="online"` (always)

3. Modify `list_machines()`: Prepend the local machine response before the DB machines list. Return `[_local_machine_response()] + [_machine_to_response(m) for m in machines]`.

4. Modify `get_machine()`: Change signature from `machine_id: UUID` to `machine_id: str`. Add early check -- if `machine_id == LOCAL_MACHINE_ID`, return `_local_machine_response()` instead of DB lookup. For remote machines, use `UUID(machine_id)` for the DB lookup.

5. Modify `scan_repos()`: Change signature from `machine_id: UUID` to `machine_id: str`. Add early check for local machine. If `machine_id == LOCAL_MACHINE_ID`, use `local_machine_manager.run_command()` to run the find command, using `settings.local_repo_scan_paths`. Return results. Skip the DB lookup and SSH connection path. For remote machines, use `UUID(machine_id)` for DB lookup.

6. Modify `get_tmux_sessions()`: Change signature from `machine_id: UUID` to `machine_id: str`. Add early check for local machine. If local, get connection from local_machine_manager. If connection exists (Docker mode), use existing `list_tmux_sessions(conn)`. If no connection (native mode), use `list_tmux_sessions_local()` from `app.local.tmux`. Return same response format. For remote machines, use `UUID(machine_id)` for DB lookup.

7. Modify `create_tmux_session()`: Change signature from `machine_id: UUID` to `machine_id: str`. Add early check for local machine. Similar routing as get_tmux_sessions -- use SSH-based or subprocess-based tmux creation based on Docker mode. For remote machines, use `UUID(machine_id)` for DB lookup.

8. Do NOT add local machine to create, update, delete, connect, disconnect, or test-connection endpoints. "This Machine" is synthetic and cannot be CRUD'd or connection-managed. Those endpoints keep their `machine_id: UUID` signatures since they only apply to remote machines. If someone tries to DELETE or PUT machine_id="local", it will naturally fail UUID validation (which is correct behavior).

**`api/sessions.py`**: Modify session creation and listing to handle local machine:

For remote machines, convert `UUID(body.machine_id)` for the DB lookup but store as `body.machine_id` string.

Since `machine_id` is now a `String(255)` column (Plan 01 changed this), remove the `UUID()` cast entirely. Change the filter to plain string comparison: `query.where(TerminalSession.machine_id == machine_id)`. This works for both "local" and UUID strings.

3. Modify `delete_session()`: When killing a session on the local machine, route tmux kill through the registry. If `machine_id == LOCAL_MACHINE_ID`:
   - Get connection via `local_machine_manager.get_connection()`
   - If connection exists (Docker mode), use existing SSH-based `kill_tmux_session(conn, tmux_name)`
   - If no connection (native mode), use `kill_tmux_session_local(tmux_name)` from `app.local.tmux`
   - The death animation: for Docker mode, use SSH to run it as before. For native mode, skip the death animation (subprocess tmux pane injection is complex and not essential). OR run it via subprocess writing to the pane TTY -- use your judgment, but don't let this block the plan. A simple approach is fine.

4. In the delete endpoint, update the `conn = await ssh_manager.get_connection(machine_id)` call to use `get_connection_for_machine(machine_id)` from the registry instead. Same for the `detach_session` endpoint.

5. In `create_session`, remove `UUID(body.machine_id)` casts when creating the TerminalSession since `machine_id` is now a string field. Just pass `machine_id=body.machine_id` directly.
