<objective>
Connect remote agent deployment to the SSH connect lifecycle and fix Claude session detection on remote machines (VPS).

Purpose: Two UAT gaps reported: (1) Claude sessions on VPS not appearing in UI, and (2) remote agent path wired but not connected. The root cause is that `get_agent_client_for_machine()` returns None for all non-local machines, and `ensure_agent()` / `deploy_agent()` are never called from the connect flow. Once remote agents are connected, Claude detection, file ops, and git ops will automatically route through the agent's faster HTTP API instead of SSH exec.

Output: Remote machines auto-deploy agent on SSH connect, agent clients stored in registry, Claude detection routes through agent for remote machines.
</objective>
