Create backend/app/services/file_service.py with these functions, all routing through run_command_on_machine from machine_registry:

1. `read_file(machine_id: str, file_path: str) -> str` - Read file content using base64 encoding for binary safety: `base64 < {safe_path}`, decode result. Reject files > 5MB (check with `stat` first). Use shlex.quote for path safety.

2. `write_file(machine_id: str, file_path: str, content: str) -> None` - Write using base64 encode/decode: `printf '%s' '{encoded}' | base64 -d > {safe_path}`. Create parent directories with mkdir -p if needed.

3. `list_directory(machine_id: str, dir_path: str) -> list[dict]` - Use `ls -1F {safe_path}` to get entries. Parse output: directories end with `/`, executables with `*`. Return list of `{"name": str, "is_dir": bool}` sorted dirs-first then alphabetical.

4. `create_file(machine_id: str, file_path: str, content: str = "") -> None` - Create parent dirs with mkdir -p, then write content (empty string if not provided).

5. `rename_file(machine_id: str, old_path: str, new_path: str) -> None` - Use `mv {safe_old} {safe_new}`.

6. `delete_file(machine_id: str, file_path: str) -> None` - Use `rm {safe_path}` (not -rf, only single files). For directory deletion use `rm -rf` but require explicit is_dir flag.

7. `file_stat(machine_id: str, file_path: str) -> dict` - Get size and mtime using `stat --format='%s %Y' {safe_path}` (Linux) with fallback to `stat -f '%z %m' {safe_path}` (macOS). Return `{"size": int, "mtime": int}`.

- `DirectoryListing(path: str, entries: list[DirectoryEntry])`
- `FileCreateRequest(machine_id: str, file_path: str, content: str = "")`
- `FileRenameRequest(machine_id: str, old_path: str, new_path: str)`
- `FileDeleteRequest(machine_id: str, file_path: str)`
- `FileStatResponse(size: int, mtime: int)`

- `AiReviewRequest(diff_text: str, custom_prompt: str | None = None)`
- `AiReviewResponse(annotations: list[ReviewAnnotation])`

Create backend/app/api/files.py with a FastAPI router (prefix="/api/files", tags=["files"]). Follow the exact pattern from api/git.py: import get_current_user for auth, use Depends for DB session.

Endpoints:

1. `GET /api/files/read` - Query params: machine_id, file_path. Returns FileContent (content + detected language + size + mtime from file_stat). Handle errors: file not found -> 404, file too large -> 413, SSH error -> 502.

2. `POST /api/files/write` - Body: FileWriteRequest. Calls write_file, returns {"success": true}. Handle permission denied -> 403.

3. `GET /api/files/list` - Query params: machine_id, dir_path. Returns DirectoryListing. Handle dir not found -> 404.

4. `POST /api/files/create` - Body: FileCreateRequest. Calls create_file. Returns {"success": true, "path": file_path}.

5. `POST /api/files/rename` - Body: FileRenameRequest. Calls rename_file. Returns {"success": true}.

6. `DELETE /api/files/delete` - Query params: machine_id, file_path. Calls delete_file. Returns {"success": true}.

7. `GET /api/files/stat` - Query params: machine_id, file_path. Returns FileStatResponse (size, mtime). Used for change detection polling.

Register the router in backend/app/api/__init__.py (follow existing pattern - add `from app.api.files import router as files_router` and include it).
