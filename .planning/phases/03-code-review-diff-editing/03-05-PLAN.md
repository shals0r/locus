Create backend/app/api/review.py with router (prefix="/api/review", tags=["review"]):

1. `GET /api/review/metadata?task_id={id}` - Look up task by ID, get linked feed_item, extract source_type and raw_payload to determine GitHub/GitLab provider info. Decrypt the credential for that source type. Call provider.get_mr_metadata(mr_id). Return MrMetadata schema. This is the key wiring: task_id -> feed_item -> source_type + raw_payload -> credential lookup -> provider -> API call.

2. `GET /api/review/diff?task_id={id}` - Same credential/provider lookup. Call provider.get_mr_diff(mr_id). Return dict with `files: list[{filename, status, patch}]`.

3. `GET /api/review/comments?task_id={id}` - Call provider.get_mr_comments(mr_id). Return list of CommentThread schemas.

4. `POST /api/review/reply` - Body: `{task_id, thread_id, body}`. Call provider.reply_to_comment(). Return the new comment.

5. `POST /api/review/submit` - Body: ReviewSubmission schema (comments, event, body). Call provider.post_review(). Return result.

6. `POST /api/review/approve?task_id={id}` - Call provider.approve(). Return result.

Helper function `_get_provider_for_task(task_id: str, db: AsyncSession)` that:
- Loads Task with feed_item relationship
- Gets source_type from feed_item
- Looks up IntegrationSource by source_type to get credential_id
- Decrypts credential using crypto.decrypt_value
- Extracts project info from feed_item.raw_payload (owner/repo for GitHub, project_id for GitLab, MR/PR number)
- Returns (provider, mr_id) tuple

Register router in backend/app/api/__init__.py.
