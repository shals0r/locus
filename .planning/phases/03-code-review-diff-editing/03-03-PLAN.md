2. Replace the single `activeDiffTab: DiffTab | null` with a proper multi-tab system:

```typescript
type TabType = "terminal" | "diff" | "editor";

interface CenterTab {
  id: string;           // unique tab ID
  type: TabType;
  label: string;
  icon: "terminal" | "diff" | "file";
  // Type-specific data
  terminalData?: { sessionId: string; machineId: string };
  diffData?: { machineId: string; repoPath: string; filePath?: string; commitSha?: string; mrId?: string; sourceType?: string };
  editorData?: { machineId: string; repoPath: string; filePath: string };
}
```

3. Add to state:
   - `tabs: CenterTab[]` - all open center panel tabs
   - `activeTabId: string | null` - currently active tab
   - `openTab(tab: CenterTab): void` - add tab if not already open (match by id), set as active
   - `closeTab(id: string): void` - remove tab, if it was active select the nearest remaining tab
   - `setActiveTab(id: string): void` - set active tab
   - `getTabById(id: string): CenterTab | undefined`
   - `reorderTabs(fromIndex: number, toIndex: number): void` - for drag-and-drop reorder

4. Keep backward compatibility: when a terminal session is created via addSession, also create a CenterTab with type "terminal". When removeSession is called, also remove the corresponding tab. When setActiveSession is called, also set the matching terminal tab as active.

5. The old `openDiffTab` should still work but now creates a CenterTab with type "diff" and a generated ID like `diff-${machineId}-${filePath || commitSha}`. `closeDiffTab` removes that tab. Keep these for backward compat with existing code that calls them.

6. Add helper: `openEditorTab(machineId: string, repoPath: string, filePath: string): void` - creates a CenterTab with type "editor", id like `editor-${machineId}-${filePath}`. If tab already exists, just activate it.
