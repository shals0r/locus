Create frontend/src/hooks/useFileOperations.ts with TanStack Query hooks:
- `useReadFile(machineId, filePath)` - GET /api/files/read?machine_id=&file_path= - Returns FileContent (content, language, size, mtime). Enabled only when both params are truthy.
- `useWriteFile()` - useMutation for POST /api/files/write. On success, invalidate BOTH the readFile query AND the diff queries. Specifically:
  - Invalidate `['file', machineId, filePath]` (the readFile cache)
  - Invalidate `['diff', machineId, repoPath]` to force the diff viewer to refetch after the file is saved. This matches the query key pattern from Plan 04's `useDiffData` hook which uses `['diff', machineId, repoPath, filePath?, commitSha?]`. Use `queryClient.invalidateQueries({ queryKey: ['diff'] })` to broadly invalidate all diff queries, or more specifically `queryClient.invalidateQueries({ queryKey: ['diff', machineId] })` if machineId is available from the mutation variables.
  - Also invalidate `['changedFiles', machineId, repoPath]` since saving a file may change the git status of that file.
  - This ensures success criterion 6 ("diff updates after save") is wired correctly.
- `useListDirectory(machineId, dirPath)` - GET /api/files/list?machine_id=&dir_path= - Returns DirectoryListing.
- `useCreateFile()` - useMutation for POST /api/files/create.
- `useRenameFile()` - useMutation for POST /api/files/rename. On success, invalidate directory listing.
- `useDeleteFile()` - useMutation for DELETE /api/files/delete. On success, invalidate directory listing and close any open editor tab for that file.
- `useFileStat(machineId, filePath)` - GET /api/files/stat?machine_id=&file_path= for change detection polling. Refetch every 5 seconds when tab is focused.

IMPORTANT: The TanStack Query keys used by useDiffData in Plan 04 follow the pattern `['diff', machineId, repoPath, ...]` and useChangedFiles uses `['changedFiles', machineId, repoPath]`. The useWriteFile onSuccess handler MUST invalidate these keys so that saving a file in the editor triggers the diff viewer to refetch fresh data. Without this wiring, the diff view would show stale content after an editor save.
