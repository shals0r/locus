<task type="auto">
  <name>Task 1: Create git service layer</name>
  <files>backend/app/services/git_service.py</files>
  <action>
Create the git service layer following RESEARCH.md Pattern 1 exactly. All git operations run via `run_command_on_machine()` from machine_registry -- NOT GitPython (per research finding: GitPython cannot work for remote repos).

Functions to implement:
- `get_repo_status(machine_id, repo_path) -> dict`: branch, is_dirty, changed_count, ahead, behind. Use `git rev-parse --abbrev-ref HEAD`, `git status --porcelain`, `git rev-list --left-right --count HEAD...@{upstream}` (wrap upstream in try/except for repos without tracking).
- `get_commit_log(machine_id, repo_path, limit=30) -> list[dict]`: NUL-separated format (`%H%x00%s%x00%an%x00%aI`) for safe parsing. Returns list of {sha, message, author, date}.
- `get_changed_files(machine_id, repo_path) -> list[dict]`: Parse `git status --porcelain`. Returns list of {status, path}. Map git status codes: M=modified, A=added, D=deleted, ??=untracked, R=renamed.
- `get_diff_for_file(machine_id, repo_path, file_path) -> str`: `git diff -- <file>` for unstaged, include `--cached` variant for staged.
- `get_commit_diff(machine_id, repo_path, commit_sha) -> str`: `git show --format='' <sha>`.
- `get_file_content_at_ref(machine_id, repo_path, ref, file_path) -> str`: `git show <ref>:<file>`.
- `git_fetch(machine_id, repo_path) -> str`: `git fetch --prune 2>&1`.
- `git_pull(machine_id, repo_path) -> str`: `git pull 2>&1`.
- `git_push(machine_id, repo_path) -> str`: `git push 2>&1`.
- `list_branches(machine_id, repo_path) -> list[dict]`: branch name + is_current using `--format='%(refname:short)%x00%(HEAD)'`.
- `checkout_branch(machine_id, repo_path, branch) -> str`: `git checkout <branch> 2>&1`.
- `create_branch(machine_id, repo_path, branch) -> str`: `git checkout -b <branch> 2>&1`.

CRITICAL: Use `shlex.quote()` for ALL user-provided values (repo_path, file_path, branch names) to prevent command injection. Import shlex at top.

For ahead/behind, wrap in try/except -- repos without a tracking upstream should return ahead=0, behind=0 without error.

  <name>Task 2: Create feed service and task service</name>

  <action>
**feed_service.py:**
- `ingest_item(db, payload: dict) -> FeedItem`: Upsert using PostgreSQL INSERT...ON CONFLICT(source_type, external_id) DO UPDATE. Use SQLAlchemy's `insert().on_conflict_do_update()` with the composite constraint. Map payload fields to FeedItem columns. Store payload's metadata as raw_payload. Use tier_hint as initial tier value (default "follow_up" if absent).
- `get_feed_items(db, include_dismissed=False, tier=None) -> list[FeedItem]`: Query with filters. Exclude snoozed items where snoozed_until > now. Order by tier priority (now=0, respond=1, review=2, prep=3, follow_up=4) then created_at desc.
- `update_feed_item(db, item_id, updates: dict) -> FeedItem`: Update is_read, is_dismissed, snoozed_until, tier.
- `snooze_item(db, item_id, until: datetime) -> FeedItem`: Set snoozed_until.
- `dismiss_item(db, item_id) -> FeedItem`: Set is_dismissed=True.
- `mark_read(db, item_id) -> FeedItem`: Set is_read=True.

Define TIER_PRIORITY dict: {"now": 0, "respond": 1, "review": 2, "prep": 3, "follow_up": 4} for sort ordering.

**task_service.py:**
- Define VALID_TRANSITIONS: queue->{active, done}, active->{done, queue}, done->set() (terminal).
- `create_task(db, data: dict) -> Task`: Create task from dict (quick promote) or with context (deep promote). Set status="queue".
- `promote_feed_item(db, feed_item_id, title, context=None, source_links=None) -> Task`: Create task linked to feed item. Copy tier from feed item.
- `transition_task(db, task_id, new_status, **kwargs) -> Task`: Validate transition against VALID_TRANSITIONS. Raise ValueError for invalid. Set started_at on queue->active (also set machine_id, repo_path, branch from kwargs). Set completed_at on *->done.
- `get_tasks(db, status=None) -> list[Task]`: Query with optional status filter. Order queue by tier priority, active by started_at, done by completed_at desc.
- `update_task(db, task_id, updates: dict) -> Task`: Update title, context.

Both services import from app.models and app.database. Use async session patterns matching existing services (auth.py).
  </action>
