**git.py** (APIRouter prefix="/api/git"):
- `GET /status?machine_id=&repo_path=` -> RepoStatus. Calls git_service.get_repo_status.
- `GET /status/all?machine_id=` -> list[RepoDetail]. Gets status for all repos on a machine (reads repos from machine's repo_scan_paths, scans them, returns status for each). Uses existing repo scanning pattern from api/machines.py.
- `GET /commits?machine_id=&repo_path=&limit=30` -> list[CommitEntry]. Calls git_service.get_commit_log.
- `GET /changed-files?machine_id=&repo_path=` -> list[ChangedFile]. Calls git_service.get_changed_files.
- `GET /branches?machine_id=&repo_path=` -> list[BranchInfo]. Calls git_service.list_branches.
- `POST /fetch` (body: {machine_id, repo_path}) -> GitOpResult. Calls git_service.git_fetch. Wrap in try/except, return success/error.
- `POST /pull` (body: same) -> GitOpResult. Same pattern.
- `POST /push` (body: same) -> GitOpResult. Same pattern.
- `POST /checkout` (body: {machine_id, repo_path, branch}) -> GitOpResult. Calls checkout_branch.
- `POST /create-branch` (body: {machine_id, repo_path, branch}) -> GitOpResult. Calls create_branch.
- `GET /diff?machine_id=&repo_path=&file_path=` -> {diff: str}. Calls get_diff_for_file.
- `GET /commit-diff?machine_id=&repo_path=&sha=` -> {diff: str}. Calls get_commit_diff.

All endpoints require auth via `Depends(get_current_user)` matching existing api routes.

**feed.py** (APIRouter prefix="/api/feed"):
- `POST /ingest` -> {status, item_id}. Webhook ingest with HMAC verification (see RESEARCH.md webhook example). Check X-Locus-Signature header; if present, verify HMAC-SHA256 against source's webhook_secret. If no signature, require JWT auth. After ingest, call broadcast_feed_update.
- `GET /` -> FeedListResponse. Query params: tier (optional), include_dismissed (bool, default false). Returns items sorted by tier priority then created_at desc.
- `GET /{item_id}` -> FeedItemResponse.
- `PATCH /{item_id}` -> FeedItemResponse. Body: FeedItemUpdate. For read/dismiss/snooze/tier changes.
- `POST /{item_id}/snooze` -> FeedItemResponse. Body: {until: datetime}. Preset calculation happens on frontend.
- `POST /{item_id}/dismiss` -> FeedItemResponse.
- `DELETE /{item_id}` -> {ok: true}. Hard delete (admin cleanup).

Register both routers in api/__init__.py and include them in main.py app.

**ws/feed.py** -- Follow the EXACT pattern from ws/status.py:
1. Module-level `feed_clients: list[asyncio.Queue] = []`
2. `broadcast_feed_update(update: dict)` function that pushes to all client queues (with try/except QueueFull)
3. WebSocket endpoint handler:
   - Auth via `?token=` query param (same as /ws/terminal and /ws/status)
   - On connect: send initial snapshot of recent unread feed items (last 50, not dismissed, not snoozed)
   - Create bounded asyncio.Queue(maxsize=100) for this client
   - Add to feed_clients list
   - Loop: await queue.get(), send JSON to client
   - Finally block: remove queue from feed_clients

Message format: `{"type": "new_item" | "item_updated" | "item_dismissed", "item_id": "uuid-str"}`
