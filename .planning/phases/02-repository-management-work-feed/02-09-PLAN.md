Install @git-diff-view/react and @git-diff-view/file:
`npm install @git-diff-view/react @git-diff-view/file`

**DiffViewer.tsx** -- Following RESEARCH.md Pattern 7:
- Props: machineId (string), repoPath (string), and one of:
  - filePath (string) for working directory diff (changed file click)
  - commitSha (string) for commit diff (timeline click)
- Uses TanStack Query to fetch diff data:
  - For file diff: GET /api/git/diff?machine_id=&repo_path=&file_path= (returns unified diff string)
  - For commit diff: GET /api/git/commit-diff?machine_id=&repo_path=&sha= (returns unified diff string)
- Parse the unified diff string and render with DiffView component
- Use `DiffModeEnum.Unified` by default (per RESEARCH recommendation: better for available width)
- Set `diffViewTheme="dark"`, `diffViewHighlight=true`, `diffViewWrap=false`
- Import and apply "@git-diff-view/react/styles/diff-view.css"
- Loading state: show skeleton/spinner while fetching
- Error state: show "Failed to load diff" message
- Empty state: "No changes" when diff is empty

Per Claude's Discretion: basic diff for Phase 2. No inline comments, no split view toggle, no annotations. Those come in Phase 3.

Use the DiffFile/generateDiffFile approach from @git-diff-view/file if the raw unified diff approach is simpler:
```
const file = generateDiffFile(oldFileName, oldContent, newFileName, newContent, lang, lang);
file.initTheme('dark');
file.init();
```
Choose whichever approach works best with the API response format. The backend returns raw diff text, so parsing it with the library's built-in parser is preferred.
