**tasks.py** (APIRouter prefix="/api/tasks"):

- `GET /` -> list[TaskResponse]. Query params: status (optional "queue"/"active"/"done"). Returns tasks sorted by tier priority within queue, started_at within active, completed_at desc within done.
- `GET /{task_id}` -> TaskResponse.
- `POST /` -> TaskResponse. Body: TaskCreate. Creates a manual task (no feed item link).
- `POST /promote` -> TaskResponse. Body: {feed_item_id: str, title: str, context: str | None, source_links: dict | None}. Calls task_service.promote_feed_item. This is the Quick Promote action.
- `POST /deep-promote` -> TaskResponse. Body: {feed_item_id: str, title: str, context: str}. Same as promote but context is LLM-generated (frontend handles LLM call). Just creates the task with richer context.
- `PATCH /{task_id}` -> TaskResponse. Body: TaskUpdate. Update title/context.
- `PATCH /{task_id}/transition` -> TaskResponse. Body: TaskTransition (status + optional machine_id/repo_path/branch for start flow). Calls task_service.transition_task. Returns 422 for invalid transitions.
- `DELETE /{task_id}` -> {ok: true}. Hard delete.

All endpoints require auth via `Depends(get_current_user)`.

Error handling: wrap transition_task ValueError in HTTPException(422, detail=str(e)) for invalid state transitions.

Register in api/__init__.py and main.py.
  </action>

  </verify>
  <done>Task API supports full lifecycle: create, promote from feed, transition (start/complete/drop), update, delete. Invalid transitions return 422.</done>

**search.py** (APIRouter prefix="/api/search"):

- `GET /?q=<query>` -> SearchResponse. Searches across multiple domains and returns grouped results:

1. **Repos**: Query machine repo_scan_paths. For each repo, check if repo name contains query (case-insensitive). Return type="repo", title=repo name, subtitle=machine name + branch, action_data={machine_id, repo_path}.

2. **Machines**: Query Machine table. Filter by name ILIKE %query%. Return type="machine", title=machine name, subtitle=status, action_data={machine_id}.

3. **Feed Items**: Query FeedItem table. Filter by title ILIKE %query% (not dismissed). Limit 10. Return type="feed_item", title=item title, subtitle=source_type + tier, action_data={item_id}.

4. **Tasks**: Query Task table. Filter by title ILIKE %query%. Limit 10. Return type="task", title=task title, subtitle=status, action_data={task_id}.

5. **Actions**: Static list of actions filtered client-side in the palette. Include a few server-side: "Toggle Sidebar", "Toggle Feed Panel", "New Task", "Promote to Task". Return type="action", title=action name, action_data={action}.

Require auth. Limit total results to 50. Results are ordered: repos first, machines, feed items, tasks, actions.

For repos, use the same pattern as the existing machines.py repo scanning -- load machines from DB, get their repos, filter by name.
