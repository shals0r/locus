**repoStore.ts** -- Zustand store (one store per domain pattern):
- State: selectedMachineId (string | null), selectedRepoPath (string | null), repos (Map<string, RepoDetail[]> keyed by machine_id), gsdStates (Map<string, GsdState> keyed by "machine_id:repo_path")
- Actions: setSelectedRepo(machineId, repoPath), clearSelection(), setRepos(machineId, repos[]), setGsdState(machineId, repoPath, gsdState)

**useGitStatus.ts** -- TanStack Query hooks:
- `useGitStatusAll(machineId)`: GET /api/git/status/all?machine_id=... with refetchInterval: 30000 (30s polling per locked decision). Returns RepoDetail[]. On success, update repoStore.
- `useCommitLog(machineId, repoPath)`: GET /api/git/commits?machine_id=&repo_path=&limit=30. Enabled only when repoPath is set.
- `useChangedFiles(machineId, repoPath)`: GET /api/git/changed-files. Enabled only when repoPath is set.
- `useBranches(machineId, repoPath)`: GET /api/git/branches.
- `useGitOp()`: useMutation for POST /api/git/fetch|pull|push|checkout|create-branch. On success, invalidate git status queries.
- `useGsdState(machineId, repoPath)`: GET /api/git/gsd-state?machine_id=&repo_path=. Enabled only when repoPath is set. Returns GsdState. On success, update repoStore.gsdStates. Use staleTime: 60000 (1 min) since GSD state changes infrequently.

**api/client.ts** -- Add helper functions if not already generic enough. Ensure apiGet/apiPost handle the new endpoints.

Use TanStack Query's useQuery/useMutation. Import queryClient for invalidation after git operations.
