**In `backend/app/ssh/tmux.py`:**

1. The `create_terminal_in_tmux` function already handles both new session creation and attach-to-existing. No changes needed to the function signature. However, add a `check_tmux_session_exists` helper:

```python
async def check_tmux_session_exists(
    conn: asyncssh.SSHClientConnection,
    session_name: str,
) -> bool:
    """Check if a tmux session with the given name exists."""
    try:
        result = await conn.run(
            f"tmux has-session -t {session_name}",
            check=True,
        )
        return True
    except asyncssh.ProcessError:
        return False
```

**In `backend/app/ws/terminal.py`:**

1. **Remove the tmux detach key sequence from the finally block.** The current code sends `\x02d` (Ctrl-B d) then calls `process.close()`. Replace the entire finally block with:

```python
finally:
    if process is not None:
        try:
            process.close()
            await asyncio.wait_for(process.wait_closed(), timeout=2.0)
        except Exception:
            pass
    logger.info("TERMINAL: Disconnected WS for session=%s (tmux persists)", session_id)
```

The key insight: `process.close()` on an asyncssh process that ran `tmux attach` does NOT kill the tmux session -- it only closes the PTY channel. The tmux session continues running in the background. The previous code was sending `\x02d` (detach) THEN closing, which was redundant. Just closing the SSH process channel is sufficient -- tmux detects the client disconnect and the session remains.

2. **Add session-exists check before creating tmux process.** Before the `create_terminal_in_tmux` call, add a check:

```python
from app.ssh.tmux import create_terminal_in_tmux, check_tmux_session_exists

# If session has a tmux name, verify it still exists on the remote
tmux_name_to_use = terminal_session.tmux_session_name
if tmux_name_to_use:
    session_exists = await check_tmux_session_exists(conn, tmux_name_to_use)
    if not session_exists:
        logger.info("TERMINAL: Tmux session %s no longer exists, will create new", tmux_name_to_use)
        tmux_name_to_use = None

process, tmux_name = await create_terminal_in_tmux(
    conn,
    session_name=tmux_name_to_use,
    working_dir=terminal_session.repo_path,
    cols=cols,
    rows=rows,
)
```

3. **Add a small delay after attach to let tmux send initial screen content** before the I/O loop begins. After `create_terminal_in_tmux` returns and before `asyncio.gather`, add:

```python
# Give tmux a moment to send initial screen redraw on reattach
await asyncio.sleep(0.1)
```

4. **Send resize after reattach** to force tmux to redraw at correct dimensions. After the sleep, add:

```python
# Force resize to sync terminal dimensions after (re)attach
try:
    process.channel.set_terminal_size(cols, rows)
except Exception:
    pass
```
