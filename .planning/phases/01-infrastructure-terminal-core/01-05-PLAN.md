    from pydantic import BaseModel, Field
    from typing import Optional

    class MachineCreate(BaseModel):
        name: str = Field(min_length=1, max_length=100)
        host: str = Field(min_length=1)
        port: int = Field(default=22, ge=1, le=65535)
        username: str = Field(min_length=1)
        ssh_key_path: str = Field(min_length=1)
        repo_scan_paths: list[str] = Field(default_factory=list)

    class MachineUpdate(BaseModel):
        name: Optional[str] = None
        host: Optional[str] = None
        port: Optional[int] = None
        username: Optional[str] = None
        ssh_key_path: Optional[str] = None
        repo_scan_paths: Optional[list[str]] = None

    class MachineResponse(BaseModel):
        id: str
        name: str
        host: str
        port: int
        username: str
        ssh_key_path: str
        repo_scan_paths: list[str]
        status: str  # "online" | "offline" | "reconnecting"

        model_config = {"from_attributes": True}

    class TestConnectionRequest(BaseModel):
        host: str
        port: int = 22
        username: str
        ssh_key_path: str

    class TestConnectionResponse(BaseModel):
        success: bool
        message: str
        tmux_sessions: list[dict] = []
    ```

    - `POST /api/machines` — Create machine from `MachineCreate`. Save to DB. Optionally auto-connect via `ssh_manager.connect()`. Return `MachineResponse`.
    - `GET /api/machines/{id}` — Get single machine with status.
    - `PUT /api/machines/{id}` — Update machine from `MachineUpdate`. If host/port/username/key changed, disconnect and reconnect.

On failure: return `{"success": false, "message": "Connection test failed: {error}"}` (per UI-SPEC copywriting).
    - `POST /api/machines/{id}/connect` — Connect to a saved machine via ssh_manager. Return status.
    - `POST /api/machines/{id}/disconnect` — Disconnect via ssh_manager. Return status.
    - `GET /api/machines/{id}/repos` — Scan repo_scan_paths on the machine via SSH (`find {path} -maxdepth 2 -name .git -type d`), return list of repo paths. Per D-16.

        machine_id: str
        session_type: str
        tmux_session_name: Optional[str]
        repo_path: Optional[str]
        is_active: bool

        model_config = {"from_attributes": True}
    ```

    **backend/app/api/sessions.py** — APIRouter prefix `/api/sessions`, all require auth:
    - `GET /api/sessions` — List all active sessions. Optional query param `machine_id` to filter.
    - `POST /api/sessions` — Create session from `SessionCreate`. Save to DB. Return `SessionResponse` with the session ID (frontend uses this to open WebSocket).
    - `DELETE /api/sessions/{id}` — Mark session as inactive (`is_active=False`). Return 204.
    - `GET /api/sessions/{id}` — Get single session.

    **backend/app/api/settings.py** — APIRouter prefix `/api/settings`, all require auth:

    Credentials sub-routes:
    - `GET /api/settings/credentials` — List all credentials. Return id, service_type, service_name (NOT the encrypted_data — never expose raw credentials in list).
    - `POST /api/settings/credentials` — Accept `{service_type, service_name, data: dict}`. Encrypt `json.dumps(data)` via `encrypt_value()`. Save Credential with encrypted_data. Return id + service_type + service_name.
    - `GET /api/settings/credentials/{id}` — Return decrypted credential data (single credential for editing). Decrypt via `decrypt_value()`, parse JSON, return.
    - `PUT /api/settings/credentials/{id}` — Update credential. Re-encrypt new data.
    - `DELETE /api/settings/credentials/{id}` — Delete credential. Return 204.
    - `POST /api/settings/credentials/{id}/test` — Test connectivity for the credential. For now, return `{"success": true, "message": "Test not implemented for this service type"}`. Will be extended per-service in Phase 4.

    Claude Code config sub-routes (per D-11, AUTH-04):
    - `GET /api/settings/claude-code` — Get Claude Code credential (service_type="claude_code"). Return type (api_key or oauth) and masked key.
    - `PUT /api/settings/claude-code` — Save/update Claude Code credential.
    - `POST /api/settings/claude-code/push/{machine_id}` — Push Claude Code auth to a machine via SSH. Connect to machine, write config to `~/.claude/` or set env vars. This enables AUTH-04.

    Status endpoint (per DEPL-03):
    - `GET /api/settings/status` — Return connected service statuses:
      ```json
      {
        "database": "connected",
        "ssh_machines": {"machine_id": "online", ...},
        "claude_code": "configured" | "unconfigured"
      }
      ```
