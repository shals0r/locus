# Phase 1 — UI Design Contract

> Visual and interaction contract for Infrastructure & Terminal Core. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (hand-built components with Tailwind CSS v4) |
| Icon library | Lucide React (lightweight, tree-shakeable, MIT licensed) |
| Font | Inter (UI) + JetBrains Mono (terminal and code) |

**Rationale:** No component library. Phase 1 has a small component surface (panels, tabs, forms, status indicators). Tailwind v4 utility classes are sufficient. Lucide provides consistent 24px stroke icons matching the engineering-tool aesthetic. Inter is the standard UI sans-serif for dark-theme tools; JetBrains Mono is purpose-built for terminal rendering.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, status dot margins |
| sm | 8px | Compact element spacing, tab padding, sidebar item padding |
| hit-area | 12px | Invisible drag handle hit area extension |
| md | 16px | Default element spacing, form field gaps, panel padding |
| lg | 24px | Section padding within settings pages, wizard step spacing |
| xl | 32px | Layout gaps between major sections |
| touch | 36px | Minimum interactive element height (desktop): tabs, sidebar items, buttons, machine tab bar |
| 2xl | 48px | Setup wizard page-level padding |
| 3xl | 64px | Login page centering offset |

**Constraints:** Touch targets for interactive elements use the declared `touch` token (36px) as minimum height. Drag handles for panel resize are 4px visible width with the declared `hit-area` token (12px) as invisible hit area for easier grabbing.

---

## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 14px | 400 (regular) | 1.5 | Inter |
| Label | 12px | 400 (regular) | 1.4 | Inter |
| Heading | 18px | 600 (semibold) | 1.3 | Inter |
| Terminal | 14px | 400 (regular) | 1.2 | JetBrains Mono |

**Constraints:**
- 2 weights only: 400 (regular) and 600 (semibold). No exceptions. Labels at 12px are already visually distinct from 14px body text — weight differentiation is not needed.
- Terminal font size is user-adjustable in a future phase; 14px is the default.
- No font sizes outside this set. If text does not fit one of these four roles, it is the wrong text.

## Panel Layout Contract

Source: D-05, D-06, D-07, D-08, Claude's discretion on proportions.

**Primary focal point:** The terminal view in the center panel. All other elements (sidebar, top bar, tab bars) are secondary and should not compete for attention at rest.

| Panel | Default Width | Min Width | Collapsed Width | Collapse Behavior |
|-------|--------------|-----------|----------------|-------------------|
| Left sidebar | 20% | 180px | 0px | Click header or keyboard shortcut |
| Center (terminals) | 80% | 400px | n/a (never collapses) |  n/a |
| Right (feed, Phase 2) | 0% (collapsed) | 240px | 0px | Not shown in Phase 1 |

**Resize handles:** 4px visible width, `border` color at rest, `accent` color on hover. `hit-area` (12px) invisible extension for easier grabbing. Double-click handle resets panel to default width (D-07).

**Top bar height:** 48px. Fixed, never scrolls.
**Machine tab bar height:** `touch` (36px). Fixed, below top bar. Scrolls horizontally if tabs overflow.
**Sub-tab bar height:** 32px. Within center panel, shows sessions for selected machine.

**Focus mode (UI-04):** Collapse sidebar via keyboard shortcut (Ctrl+B). Center panel expands to full width. Right panel remains collapsed in Phase 1.

## Interaction Patterns

### Panel Resize
- Drag handle to resize panels continuously
- Double-click handle to reset to default proportions (D-07)
- Keyboard: Ctrl+B toggles sidebar collapse (UI-04 focus mode)

### Machine Tab Navigation
- Click machine tab to switch active machine
- Click "+" to add new machine (opens form)
- Right-click machine tab for context menu: Rename, Settings, Disconnect

### Session Tab Navigation
- Click session tab to switch active session
- Click "+" on session tab bar to open repo picker (D-04)
- Middle-click or click X to close session tab (tmux session persists remotely)
- Claude Code tabs show a pulsing amber dot when waiting for input (TERM-07)

### SSH Connection Lifecycle
- On machine connect: auto-detect tmux sessions (D-14), show picker if multiple exist
- On disconnect: show reconnecting banner with spinner
- On reconnect: auto-reattach to previous tmux session (D-13)
- On max retries exhausted: show failed banner with manual "Reconnect" button (D-13)

### First-Run Setup
- Step 1 (required): Set password
- Step 2 (required): Add at least one machine with test connection
- Step 3 (optional): Add service credentials, skip button available (D-09)
- Progress indicator shows completed/remaining steps
