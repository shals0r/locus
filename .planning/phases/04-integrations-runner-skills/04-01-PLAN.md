**WorkerSupervisor service** (`backend/app/services/worker_supervisor.py`):

Build a singleton `WorkerSupervisor` class using `asyncio.create_subprocess_exec()`. Follow the research Pattern 1 architecture:

```python
import asyncio
import json
import logging
import os
from dataclasses import dataclass, field
from enum import Enum
from typing import Optional
from app.database import async_session_factory
from app.services.crypto import decrypt_value

logger = logging.getLogger(__name__)

class WorkerState(str, Enum):
    STOPPED = "stopped"
    STARTING = "starting"
    RUNNING = "running"
    DEGRADED = "degraded"
    CRASHED = "crashed"
    DISABLED = "disabled"
    STOPPING = "stopping"

@dataclass
class WorkerProcess:
    worker_id: str
    process: Optional[asyncio.subprocess.Process] = None
    state: WorkerState = WorkerState.STOPPED
    failure_count: int = 0
    log_buffer: list[str] = field(default_factory=list)
    _monitor_task: Optional[asyncio.Task] = None
    _log_task: Optional[asyncio.Task] = None
```

Key methods to implement:
1. `async def start_worker(self, worker_id: str, script_path: str, env: dict, venv_python: str | None = None) -> None` -- spawns subprocess via `asyncio.create_subprocess_exec(python, script_path, stdout=PIPE, stderr=STDOUT, env=env)`, creates monitor task
2. `async def stop_worker(self, worker_id: str) -> None` -- sets state to STOPPING, calls `proc.terminate()`, waits up to 5 seconds, then `proc.kill()` as fallback
3. `async def restart_worker(self, worker_id: str) -> None` -- stop then start with same config
4. `async def _monitor_worker(self, wp: WorkerProcess, script_path: str, env: dict, venv_python: str | None) -> None` -- waits for process exit, implements exponential backoff restart (base=2s, max=300s), disables after MAX_FAILURES=5, posts feed notification via ingest API on disable
5. `async def _read_logs(self, wp: WorkerProcess) -> None` -- reads `async for line in proc.stdout`, appends to ring buffer (LOG_BUFFER_SIZE=100), broadcasts to subscribers via `asyncio.Queue`
6. `def subscribe_logs(self, worker_id: str, queue: asyncio.Queue) -> None` and `def unsubscribe_logs(self, worker_id: str, queue: asyncio.Queue) -> None`
7. `def get_worker(self, worker_id: str) -> WorkerProcess | None`
8. `def get_all_workers(self) -> dict[str, WorkerProcess]`
9. `async def build_worker_env(self, worker_id: str) -> dict` -- decrypts credentials from DB using `decrypt_value()`, adds `LOCUS_INGEST_URL=http://localhost:8080/api/feed/ingest`, generates `LOCUS_WORKER_SECRET` (container-scoped HMAC secret), injects `POLL_INTERVAL`, and per-credential env vars
10. `async def shutdown(self) -> None` -- terminates all workers, used by FastAPI lifespan

Constants: `MAX_FAILURES = 5`, `BACKOFF_BASE = 2`, `BACKOFF_MAX = 300`, `LOG_BUFFER_SIZE = 100`

The supervisor maintains a module-level `_worker_secret` generated once at import via `secrets.token_hex(32)`. All workers share this HMAC secret for authenticating with the ingest API.

`async def setup_venv(self, worker_id: str, requirements_path: str) -> str`:
- Creates venv at `/data/workers/user/{worker_id}/.venv/` using `asyncio.create_subprocess_exec("python3", "-m", "venv", venv_path)`
- Runs `pip install -r requirements.txt` in the venv
- Returns path to venv python binary
