# Phase 4 — UI Design Contract

> Visual and interaction contract for Integrations Runner & Skills. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (hand-built components with Tailwind CSS v4, continuing Phase 1 pattern) |
| Icon library | Lucide React (`lucide-react ^0.500.0`, already installed) |
| Font | Inter (UI) + JetBrains Mono (terminal, code, log output) |

**Rationale:** Continues the established pattern from Phases 1-3. No component library needed. Phase 4 reuses existing layout patterns (SettingsPage sections for worker cards, ReviewChat side panel for Integrator chat, sidebar chips for skills). All new components follow the same Tailwind v4 utility class approach with project CSS custom properties.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, status dot margins, inline padding |
| sm | 8px | Compact element spacing, chip padding, worker card internal gaps, skill chip row gap |
| hit-area | 12px | Resize handle hit area (Integrator panel edge). Smallest multiple of 4 that provides reliable drag target. |
| md | 16px | Default element spacing, form field gaps, worker card padding |
| lg | 24px | Section padding within Settings Integrations section |
| xl | 32px | Gap between worker cards |
| touch | 36px | Minimum interactive element height: skill chips, play/pause buttons, Deploy button. WCAG 2.5.5 minimum touch target guidance. |
| 2xl | 48px | Page-level padding in Settings view |
| 3xl | 64px | Not used in Phase 4 |

Exceptions:
- 12px (`hit-area`) — included as the smallest practical drag-target width; a multiple of 4 but below the standard 16px step. Used exclusively for the Integrator panel resize handle.
- 36px (`touch`) — included per WCAG 2.5.5 enhanced touch target guidance (minimum 44px ideal, 36px acceptable for desktop-only single-user app). Used for interactive element minimum heights.

## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 14px | 400 (regular) | 1.5 | Inter |
| Label | 12px | 400 (regular) | 1.4 | Inter |
| Heading | 18px | 600 (semibold) | 1.3 | Inter |

| Chat message | 14px | 400 (regular) | 1.5 | Inter |
| Chat code block | 12px | 400 (regular) | 1.3 | JetBrains Mono |

**Constraints:**
- 2 weights only: 400 (regular) and 600 (semibold). Consistent with Phase 1 contract.
- Log output in WorkerLogPanel uses JetBrains Mono at 12px to match terminal aesthetic while being more compact.
- Integrator chat messages use Body role (Inter 14px). Code blocks within messages use Chat code block role (JetBrains Mono 12px), matching the established ReviewChat.tsx pattern.
- Worker card metadata (poll interval, last poll time, items ingested) uses Label role (12px).

## Focal Point

The primary screen for Phase 4 is the Settings > Integrations section. The focal point is the **worker card list** -- the user's eye should land on the first worker card (or the empty state CTA) immediately upon scrolling to the Integrations section. The "New Integration" button is positioned top-right of the section heading as the primary action entry point.

**Layout structure (top to bottom):**

2. **Messages area** (flex-1, scrollable): Chat bubbles + structured cards, auto-scroll to bottom
3. **Credential feedback bar** (conditional, 36px): Shows "Credentials saved securely -- Claude never sees your credentials." when credentials are stored during the flow

**Machine selector:** Dropdown in the header showing connected machines with Claude Code installed. Required because the Integrator runs Claude Code CLI on a specific machine (D-12). If no machine has Claude Code, show disabled state with tooltip "No machines with Claude Code detected."

## Integrator Structured Cards

Source: D-14, D-15; discretion on card design

Structured cards appear inline within the chat message flow. They are visually distinct from chat bubbles.

### Config Step Card
- Border: `border-border` with left accent bar (`border-l-4 border-l-accent`)
- Background: `bg-secondary`
- Content: Step label (Label role) + input field or credential reference
- Action: "Apply Config" button (accent, small)

### Dry-Run Test Card
- Border: `border-border`
- Background: `bg-secondary`
- States:
  - **Running:** Pulsing success dot + "Testing against live API..."
  - **Success:** Success dot + "Returned {N} items" + expand arrow to show preview cards
  - **Failed:** Error dot + error message + "Claude will iterate on the code."

### Preview Feed Item Card
- Mimics FeedCard layout: left tier-color bar, title, snippet, source icon
- Background: `bg-dominant/50` (slightly recessed to differentiate from real feed items)
- Non-interactive (display only, no triage actions)

### Deploy Card
- Full-width within chat
- "Deploy Worker" button: accent background, white text, `touch` height (36px), full width
- States:
  - **Ready:** Button enabled, label "Deploy Worker"
  - **Deploying:** Button disabled, spinner, label "Deploying..."
  - **Deployed:** Button replaced with success message "Worker deployed and running" + success dot

## Worker Card Layout

Source: D-20, D-21; pattern: SettingsPage sections

Each worker renders as a card within the Integrations section of SettingsPage.

**Card structure:**
```
+---------------------------------------------------+
| [status dot] Worker Name          [gear] [play/pause] |
| Poll: 5m  |  Last: 2m ago  |  Items: 1,247         |
+---------------------------------------------------+
| [expandable log panel - click card to toggle]       |
| > 2026-03-28 14:32:01 INFO  Polled 12 items        |
| > 2026-03-28 14:32:01 INFO  Posted to ingest API   |
| > 2026-03-28 14:31:58 ERROR Connection timeout      |
|                                     [Load more]     |
+---------------------------------------------------+
```

**Card dimensions:**
- Padding: `md` (16px)
- Border: `border border-border rounded-lg`
- Background: `bg-secondary`
- Gap between cards: `lg` (24px) -- using flex gap in parent container
- Max width: `max-w-2xl` (consistent with existing SettingsPage)

**Top row:**
- Status dot (8px, color per status table) + Worker name (Body, 14px, semibold) + spacer + Gear icon button (16px, muted, hover:primary-text) + Play/Pause icon button (16px, muted, hover:primary-text)
- Play icon: `Play` from lucide-react (for stopped/crashed/disabled workers)
- Pause icon: `Pause` from lucide-react (for running workers)
- Gear icon: `Settings2` from lucide-react

**Metadata row:**
- Label role (12px, muted): "Poll: {interval}" | "Last: {relative_time}" | "Items: {count}"
- Pipe separators in muted color
- If worker is disabled: additional "Disabled after {N} failures" in error color

**Log panel (expanded state):**
- Click anywhere on the card (except buttons) to toggle log expansion
- Background: `bg-dominant` (recessed from card)
- Border top: `border-t border-border`
- Max height: 200px, scrollable
- Log lines: Log role (JetBrains Mono 12px, line-height 1.3)
- Log line format: `{timestamp} {LEVEL} {message}`
- Level coloring: INFO = muted, WARN = warning, ERROR = error
- Auto-scroll to bottom when new lines arrive (unless user has scrolled up)
- "Load earlier logs" link at top of log area (Label role, accent color)

## Worker Quick Config Popover

Source: D-17

Triggered by gear icon on WorkerCard. Renders as a dropdown/popover anchored to the gear button.

**Popover structure:**
- Width: 280px
- Background: `bg-dominant`
- Border: `border border-border rounded-lg shadow-lg`
- Padding: `md` (16px)

**Fields:**
1. **Poll Interval:** Label "Poll every" + number input + "minutes" suffix. Default from current worker config.
2. **Credential:** Label "Credential" + dropdown of stored credentials (from existing credential store). Shows current credential name.
3. **Enabled:** Label "Enabled" + toggle switch. When toggled off, worker stops. When toggled on from disabled state, resets failure count and restarts.
4. **Apply Config button:** "Apply Config" (accent, small). Closes popover on save.

Source: D-22, D-23, D-24

**Layout:**
- Horizontal wrapping chip row with `gap-2` (8px)
- Overflow: wrap to next line (sidebar is narrow, chips will wrap naturally)

**Skill Chip:**
- Background: `bg-secondary`
- Border: `border border-border rounded-md`
- Padding: `px-2 py-1` (horizontal 8px, vertical 4px)
- Text: Label role (12px, primary-text)
- Hover: `bg-hover border-muted`
- Active (session running): `border-accent bg-accent/10 text-accent`
- Icon: `Terminal` icon (12px) before chip text when in active state
- Click: Opens Claude Code terminal session pre-loaded with skill command (D-25)

**Empty state:** When repo has no `.claude/commands/` or `.claude/skills/`, SkillBar is not rendered at all (D-24 says "only visible when a repo is selected and has skills").

**Loading state:** Single chip-sized skeleton shimmer while SSH scan completes.

## Interaction Patterns

### Worker Management

- **View workers:** Navigate to Settings page, scroll to Integrations section. Worker cards are listed vertically.
- **Start/stop worker:** Click play/pause button on worker card. No confirmation for start. Stop shows a brief confirmation tooltip: "Stop {name}?" with confirm/cancel.
- **Expand logs:** Click anywhere on worker card body (not on buttons). Log panel slides open with 200ms transition. Click again to collapse.
- **Quick config:** Click gear icon. Popover appears anchored to icon. Click outside or "Apply Config" to close.
- **Re-enable disabled worker:** Click "Re-enable Worker" button shown on disabled worker cards (replaces play/pause). Resets failure count and starts worker.
- **Delete worker:** Right-click or long-press on worker card shows context menu with "Delete" in destructive color. Confirmation dialog required.
- **New integration:** Click "New Integration" button at top of Integrations section. Opens Integrator chat panel.

### Integrator Chat Flow

Source: D-14

1. User clicks "New Integration" -- Integrator panel slides in from right edge (300ms, ease-out).
2. User types description of what to connect (e.g., "Connect to my company's GitLab and watch for merge request reviews assigned to me").
3. Claude responds with plan + asks for credentials if needed.
4. Credential prompt appears as structured card. User enters credentials via Locus credential form (existing CredentialForm pattern). Card updates to show "Credentials saved securely -- Claude never sees your credentials."
5. Claude writes worker code and initiates dry-run. Test card shows "Testing against live API..." with pulsing indicator.
6. On success: test card shows item count + expandable preview cards mimicking feed items.
7. On failure: test card shows error. Claude responds with adjusted code and retries.
8. When ready: Deploy card appears with full-width "Deploy Worker" button.
9. User clicks Deploy. Button shows deploying state. On success, card updates to "Worker deployed and running."
10. Panel can be closed. Reopening for the same worker resumes conversation (D-16).

**Keyboard shortcut:** None assigned (Integrator is launched from Settings, not a frequent action).

### Skill Execution

Source: D-24, D-25

1. User selects a repo in the sidebar (existing interaction).
2. If repo has skills, SkillBar appears below repo info.
3. User clicks a skill chip (e.g., "commit").
4. A new Claude Code terminal session opens in the center panel, pre-loaded with the skill command (e.g., `/commit`).
5. User interacts with Claude in the terminal as normal.
6. While session is active, the skill chip shows active state (accent border/text).

### Integrator Panel Resize

- Same pattern as ReviewChat.tsx: drag left edge to resize.
- Min 320px, max 640px, default 420px.
- Double-click resize handle: reset to 420px default.

## Accessibility Notes

- Worker status dots include `title` attributes with the status label for screen readers.
- Skill chips are `<button>` elements with accessible labels: `aria-label="{skill_name}: {description}"`.
- Log panel uses `role="log"` and `aria-live="polite"` for screen reader announcements of new log lines.
- Integrator chat messages area uses `role="log"` and `aria-live="polite"`.
- All icon-only buttons (gear, play/pause, close) have `title` and `aria-label` attributes.
- Color is never the sole indicator of state: status dots are paired with text labels, log levels are prefixed with text (INFO/WARN/ERROR).
