**Integrator service** (`backend/app/services/integrator_service.py`):

Follow research Pattern 3. Manages Claude Code CLI sessions for the Integrator chat:

```python
import asyncio
import json
import logging
import os
from dataclasses import dataclass, field
from datetime import datetime, timezone
from typing import Optional

logger = logging.getLogger(__name__)

INTEGRATOR_SYSTEM_PROMPT = """You are the Locus Integrator, a specialized assistant that helps users build integration workers for Locus.

Your job:
1. Understand what external service the user wants to connect
2. Ask for any configuration details (API endpoints, project IDs, etc.)
3. Write a standalone Python worker script following the Locus worker contract
4. Test the worker by running its poll() function once
5. Help deploy the worker to the Locus runner

Worker contract:
- Standalone Python script with #!/usr/bin/env python3
- Reads config from environment variables: LOCUS_INGEST_URL, LOCUS_WORKER_SECRET, POLL_INTERVAL, plus service-specific vars
- Defines a poll() -> list[dict] function returning items with: source_type, external_id, title, snippet, url, tier_hint, source_icon, metadata
- Main loop calls poll() on POLL_INTERVAL, POSTs results to LOCUS_INGEST_URL with X-Locus-Worker-Secret header
- Uses httpx for HTTP requests
- Prints structured logs to stdout: {ISO_TIMESTAMP} {LEVEL} {message}
- All print statements use flush=True

IMPORTANT:
- NEVER ask for or handle credentials directly. Tell the user credentials are managed by Locus.
- Write worker scripts to /data/workers/user/{worker_id}/worker.py
- If the worker needs extra pip packages, write a requirements.txt alongside it
- When testing, run: python -c "import sys; sys.path.insert(0, '.'); from worker import poll; items = poll(); print(f'Got {len(items)} items'); [print(f'  - {i.get(\"title\",\"?\")}') for i in items[:5]]"
"""

@dataclass
class IntegratorConversation:
    session_id: str  # Claude Code CLI session ID
    machine_id: str
    worker_id: str | None = None
    created_at: datetime = field(default_factory=lambda: datetime.now(timezone.utc))

class IntegratorService:
    def __init__(self):
        self._conversations: dict[str, IntegratorConversation] = {}

    async def send_message(
        self,
        conn,  # asyncssh connection
        message: str,
        session_id: str | None = None,
        cwd: str = "/data/workers",
    ) -> dict:
        """Send a message to Claude Code CLI and get structured response.

        Uses -p (print mode) with --output-format json for structured output.
        Uses --resume to maintain conversation context across messages.
        """
```

Implementation of `send_message`:
1. Build command: `claude -p --output-format json --append-system-prompt '{INTEGRATOR_SYSTEM_PROMPT}' --allowedTools 'Read,Edit,Bash,Write'`
2. If `session_id` is provided: add `--resume {session_id}`
3. Pipe the user message via stdin: `echo '{escaped_message}' | cd {cwd} && {cmd}`
4. Execute via `conn.run()` with timeout=120 seconds
5. Parse JSON response from stdout
6. Extract: `result` (text response), `session_id` (for continuation)
7. Parse response text for structured card indicators:
   - If response mentions credentials needed: add `{"type": "credential_prompt", "service": "..."}` to structured_cards
   - If response mentions test/dry-run results: add `{"type": "test_result", "success": bool, "item_count": int, "items": [...]}` to structured_cards
   - If response mentions worker is ready to deploy: set `worker_ready=True` and add `{"type": "deploy_ready", "script_path": "..."}` to structured_cards
8. Return `IntegratorResponse` dict

```typescript
import { create } from "zustand";

interface ChatMessage {
  id: string;
  role: "user" | "assistant";
  content: string;
  structured_cards?: StructuredCard[];
  timestamp: string;
}

interface StructuredCard {
  type: "credential_prompt" | "config_step" | "test_running" | "test_success" | "test_failed" | "deploy_ready" | "deploy_complete";
  data: Record<string, unknown>;
}

interface IntegratorStore {
  isOpen: boolean;
  messages: ChatMessage[];
  sessionId: string | null;
  machineId: string | null;
  workerId: string | null;  // Non-null when editing existing worker
  loading: boolean;  // Waiting for Claude response
  deploying: boolean;
  deployed: boolean;
  availableMachines: Array<{ id: string; name: string }>;

  open: (workerId?: string) => void;  // Open panel, optionally for editing existing worker
  close: () => void;
  setMachineId: (id: string) => void;
  sendMessage: (content: string) => Promise<void>;
  deploy: (scriptPath: string, name: string, sourceType: string) => Promise<void>;
  fetchMachines: () => Promise<void>;
  reset: () => void;  // Clear conversation for new integration
}
```

`sendMessage`: calls `POST /api/integrator/message` with `{ content, session_id, machine_id, worker_id }`. Appends user message to `messages` immediately, then appends assistant response when received. Updates `sessionId` from response.

`deploy`: calls `POST /api/integrator/deploy`. Sets `deploying=true`, then `deployed=true` on success.

`open`: if called from "New Integration" button, `workerId` is null. If called from worker card edit, `workerId` is set. Fetches machines.

Listen for `open-integrator` custom event (dispatched by IntegrationSettings "New Integration" button): `window.addEventListener("open-integrator", ...)`.

**useIntegratorChat hook** (`frontend/src/hooks/useIntegratorChat.ts`):

Simple wrapper that provides `sendMessage` with auto-scroll and error handling:

```typescript
export function useIntegratorChat() {
  const { sendMessage, loading } = useIntegratorStore();
  const messagesEndRef = useRef<HTMLDivElement>(null);

  const send = useCallback(async (content: string) => {
    await sendMessage(content);
    messagesEndRef.current?.scrollIntoView({ behavior: "smooth" });
  }, [sendMessage]);

  return { send, loading, messagesEndRef };
}
```
